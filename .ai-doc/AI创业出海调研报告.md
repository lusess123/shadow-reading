## 三、方向分析（与 Codex 协作讨论）

| 方向 | 推荐度 | 原因 |

| 方向 | 推荐度 | 原因 |

| AI 内容创作工具 | ⭐⭐⭐ | 不发挥技术优势 |

### 3.3 垂直场景细分选择

| 场景 | 推荐度 | 优势 | 劣势 |
|------|--------|------|------|
| **水管工 (Plumber)** | 🥇 | 成功案例多，ROI 明确 | 市场相对小 |
| HVAC（暖通空调）| 🥈 | 市场更大 | 竞争稍强 |

**最终选择**：水管工（Plumber）→ HVAC → 电工

## 七、技术方案

### 7.1 技术栈

```
你的 Voice Agent 技术（核心！）
    ↓
LiveKit 电话集成
├── Phone Numbers（美国号码）
├── SIP Trunk（Twilio/Telnyx）
└── Voice AI SDK
    ↓
后端服务
├── Supabase（Auth + DB + Functions）
└── Next.js（Dashboard）
    ↓
第三方集成
├── Jobber/Housecall Pro API（日历）
├── Stripe/Lemon Squeezy（支付）
└── Vercel（部署）
```

### 7.2 LiveKit 参考示例

**Doheny Surf Desk Booking Agent** - 完美参考！

```
doheny-surf-desk/
├── agent.py                    # 主入口
├── agents/
│   ├── frontdesk_agent.py      # 前台问候 & 路由
│   ├── intake_agent.py         # 信息收集
│   ├── scheduler_agent.py      # 预约管理
│   ├── billing_agent.py        # 支付处理
│   └── observer_agent.py       # 背景监控
├── tasks/                      # 任务模块
└── tools/                      # 工具函数
```

GitHub: https://github.com/livekit-examples/python-agents-examples/tree/main/complex-agents/doheny-surf-desk

### 7.3 核心对话场景

1. **紧急情况**："I have a leak / My pipe burst"
   - 优先级：最高
   - 流程：确认地址 → 预估到达时间 → 派单

2. **报价咨询**："How much to fix a toilet?"
   - 流程：了解问题 → 给出范围 → 安排上门评估

3. **预约服务**："I need to schedule a plumber"
   - 流程：了解需求 → 查看日历 → 确认时间 → 发送提醒

### 10.1 主要风险

| 风险 | 可能性 | 影响 | 应对策略 |

### 10.2 Pivot 信号

如果出现以下情况，考虑调整方向：
- Month 3 还没有任何付费客户
- 客户普遍反馈"不需要这个"
- 竞品推出免费版
- 你自己失去热情

### 10.3 备选方向

1. **HVAC（暖通空调）** - 类似场景，更大市场
2. **电工** - 同样的技术栈
3. **房产经纪人** - 高客单价
4. **医疗预约** - 市场更大，但合规复杂
